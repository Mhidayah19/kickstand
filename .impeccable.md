# Kickstand Design Context

## Design Context

### Users
Singapore motorcycle owners who take pride in their machines. They're logging maintenance, tracking compliance, and managing their garage — typically in short sessions while thinking about their bike. They want to feel like the app matches the quality of what they ride.

### Brand Personality
**Kickstand** — mechanical precision meets high-end editorial sophistication. Design system theme: "The Precision Atelier".
Three words: **Controlled. Precise. Considered.**

Emotional goal: **calm and control** — the interface should feel like a well-maintained machine. No noise, no clutter. Everything in its place.

### Aesthetic Direction
- **Visual tone:** Modern minimalist with editorial restraint. Asymmetric layouts, bold typography, deliberate whitespace.
- **Theme:** Light mode only (dark mode planned for future — design tokens should support it).
- **References:** The stitch mockups in `/stitch/` are the canonical visual reference. The `modern_minimalist_mechanic/DESIGN.md` is the design system bible.
- **Anti-references:** Generic SaaS blue/white, centered everything, divider lines between list items, drop shadows, playful/rounded bubbly UI.

### Color Palette
| Token | Value | Use |
|-------|-------|-----|
| `charcoal` | `#1E1E1E` | Headings, structural weight, primary CTAs |
| `yellow` | `#F2D06B` | High-priority actions, brand moments — use sparingly |
| `sand` | `#C7B299` | Secondary text, warmth, large background sections |
| `surface` | `#F9F9F9` | Main canvas |
| `surface-low` | `#F3F3F3` | Subtle background sections, input fields |
| `surface-card` | `#FFFFFF` | Cards, elevated surfaces |
| `outline` | `#D0C5BA` | Ghost borders only, never full opacity |
| `danger` | `#DC2626` | Errors, alerts |
| `success` | `#22C55E` | Positive states |

### Typography
**Font:** Plus Jakarta Sans exclusively — never Regular (400). Minimum weight is Medium (500).

| Use | Weight | Style |
|-----|--------|-------|
| Page headlines | Bold 700 / ExtraBold 800 | Large, tight letter-spacing |
| Card titles | Bold 700 | Standard |
| Body copy | Medium 500 | Normal spacing |
| Labels / metadata | Medium 500 | All-caps, wide tracking (`tracking-wide-1` or `tracking-widest`) |

### Display typography (countdown numbers)

Display sizes are reserved for **the single hero number on a screen** — the giant countdown on the home hero pedestal, and the lifetime-spend figure on category drill-downs. Never use display sizes for running text.

| Token | Size | Use |
|-------|------|-----|
| `text-display-sm` | 72px | Low-confidence / fallback predictions, category subtotals |
| `text-display-md` | 96px | Medium-confidence predictions, category detail lifetime spend |
| `text-display-lg` | 120px | High-confidence on sub-screens |
| `text-display-xl` | 148px | Home hero countdown — one per screen, only place this size appears |

All display sizes use `font-sans-xbold` (800) and require `style={{ fontVariant: ['tabular-nums'] }}` on the `<Text>` so digits have equal width.

### Design Principles
1. **Calm through clarity** — Every element earns its place. Remove before adding. Whitespace is not wasted space.
2. **Separation without lines** — Never use dividers. Use background tonal shifts or vertical whitespace to create hierarchy.
3. **Precision in detail** — Labels are all-caps with wide tracking. Corners are generously rounded (24–32px). Inputs have a focused yellow accent bar, not a border.
4. **Glassmorphism for floating elements** — Headers and tab bars use `expo-blur` with semi-transparent overlay. Blur intensity 70–80.
5. **Yellow is precious** — The brand yellow exists to draw the eye to one thing per screen. Don't dilute it by using it everywhere.
6. **Support future dark mode** — Use semantic color tokens, not hardcoded values. Every color decision should have a logical dark-mode equivalent.

### Component Patterns
- **Cards:** `rounded-2xl` (24px), `bg-surface-card`, no border, ambient shadow at 4–6% opacity
- **Buttons:** Primary = yellow + charcoal text. Secondary = charcoal + white text. Tertiary = no fill. All `rounded-full`.
- **Inputs:** `bg-surface-low`, no border, yellow 2px underline bar on focus
- **Badges/chips:** All-caps label text, wide tracking, 4 variants: yellow / danger / charcoal / surface
- **Status indicators:** Use `PillBadge` component — never raw colored text
- **Card backgrounds:** default to `bg-sand/10` for all large container cards. Reserve pure-white `bg-surface-card` for small overlay controls (nav bell, dismiss X) that need to pop against warm cards.
- **Icon wells inside cards:** `bg-sand/20` (one step warmer than the card around them)
- **Input fields:** `bg-sand/15` with a 2px yellow underline on focus (existing rule, unchanged)
- **Muted text:** `text-charcoal/55` — no dedicated token, use opacity modifier on charcoal
- **Eyebrow micro-labels:** the small uppercase label that sits above section headers (e.g. "NEXT UP" above "Upcoming services") uses `text-[10px] font-sans-bold tracking-atelier uppercase text-charcoal/55`
